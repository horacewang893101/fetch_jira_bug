# MP-321: “See More” redirects without preserving filter context

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Location:* Home page (Dashboard) - “See More” button  
*Severity:* Medium  
*Impact:* Users lose browsing context.

*Description:*
After clicking “See More” in the agents section, the user is redirected and may lose the selected category context.

*Reproduction Steps:*

# Select a category (e.g., “Automobile”)
# Click “See More”
# Check whether the category is preserved

*Evidence:* 

* Screenshot: [dashboard-see-more-clicked.png]

*Recommendation:* 

* Preserve filter parameters during navigation
* Pass context via URL query parameters
