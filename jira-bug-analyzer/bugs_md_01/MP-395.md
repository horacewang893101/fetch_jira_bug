# MP-395: [Recommendation] Celery service - Scale horizontally and reduce per-pod size

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
Requests: *1000m / 8 GiB*, limit: *12 GiB*; observed: *~6m / ~2.6 GiB*.
Oversized per-pod reservations constrain scheduling and node choices.

*Impact:*
Inefficient node utilization and higher cost; risk of unnecessary node scale-ups.

*Recommended Fix:*

* *CPU request:* 100–200m
* *Memory request:* 2–4 GiB
* *Memory limit:* 6 GiB
Prefer horizontal scaling via replicas/HPA; validate CPU as a throughput proxy (consider KEDA).
