# MP-274: Database Credentials Exposed in Logging Configuration (Secret: serilogsettings)

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Description*
The Serilog configuration stored in Key Vault secret {{serilogsettings}} contains a PostgreSQL connection string with *superuser credentials* ({{User Id=postgres; Password=postgres}}). This is used for logging sinks, but exposes full DB access.

*Impact*
If the secret is compromised, attacker gains unrestricted access to the database, enabling data exfiltration, schema modification, and persistence. Logging sinks should never use privileged accounts.

*Evidence (masked)*
{{connectionString="Server=serviceme-pg9gudls.postgres.database.azure.com;Database=serviceme-db;Port=5432;User Id=postgres;Password=********;"}}



*Recommended Fix*

* Use a *least-privileged database account* dedicated to logging (INSERT-only permissions on log table).
* Rotate credentials immediately; remove {{postgres}} usage.
* Store DB credentials as a separate Key Vault secret, not inside logging config.
