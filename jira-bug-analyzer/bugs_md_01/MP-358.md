# MP-358: Missing Accessible Names (aria-label) for Buttons Across the Entire Application

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Priority*: Critical  
*Impact*: Accessibility (WCAG 2.1 AA - 4.1.2), Screen Reader Support  
*Category*: Name, Role, Value

*Problem*:
Nearly all icon-only buttons in the application do not have properly defined accessible names (aria-label). Screen readers cannot describe what these buttons do, making the application unusable for blind and low-vision users.

*Affected Elements*:

* *Application header*: theme, notifications, settings (icons without text)
* *Login page*: show/hide password (eye icon), language switch (globe icon)
* *Sidebar navigation*: expand/collapse controls
* *Agent cards*: action buttons (edit, delete, copy)
* *Forms*: helper buttons (e.g., "?" icons)
* *Pagination*: navigation arrows
* *Modals*: close button (X icon)
* *Toolbars and panels*: multiple icon buttons without labels

*Evidence*: DOM snapshot analysis shows unnamed buttons in many places:

*Recommended Fix*:
Run an audit of all icon-only buttons and add appropriate labels:

<!-- Login -->

<!-- Pagination -->

<button aria-label="Next page"><svg>...</svg></button>

<!-- Modals -->
