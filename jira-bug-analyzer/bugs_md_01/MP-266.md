# MP-266: JWT validation misconfiguration in Knowledge workload settings

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Description*
JWT validation is misconfigured in {{appsettings}} secret stored in Key Vault used by Knowledge workload: expiry checks are disabled ({{CheckExp=false}}), and app tokens live for ~180 days ({{ExpireMinutesForApp=259200}}). Secrets are static and unrotated.



*Impact*
Attackers can mint/reuse tokens indefinitely, escalate privileges, and bypass session controls.



*Evidence*
{{AuthenticationConfigs.JwtConfigs.CheckExp=false}}
{{ExpireMinutesForApp=259200}}
{{JwtSecrets=[...]}}



*Recommended Fix*

* Set {{CheckExp=true}}.
* Reduce token lifetimes (e.g., 15â€“60 min access tokens).
* Use *RS256/ES256* with key in Key Vault; implement *key rotation* and {{kid}}.
* Enforce refresh token rotation & revocation.
