# MP-360: Insufficient Color Contrast Across the Entire Application

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Priority*: High  
*Impact*: Accessibility (WCAG 2.1 AA - 1.4.3), Readability  
*Category*: Contrast (Minimum)

*Problem*:
In multiple areas of the application, color contrast is insufficient. This reduces readability, especially for users with low vision.

*Affected Elements*:

* Disabled buttons (e.g., pagination) – text and icons hard to distinguish from the background
* "Pre-built" badges on agent cards
* Form validation messages (red text on pink/light background, e.g., "Username cannot be empty")
* Placeholder text in inputs – too low contrast against the background
* Various labels and helper texts in UI panels

*Evidence*:

* Login form: error messages in red (#FF4D4F) on a pink background do not meet the required 4.5:1 contrast ratio
* "Username" and "Password" placeholders appear low-contrast on white input backgrounds

*Recommended Fix*:

# Perform a contrast audit for all states (disabled, error, placeholder, labels)
# Ensure text meets at least 4.5:1 (normal text) or 3:1 (large text)
# Validate using tools like the WebAIM Contrast Checker

{code:css}/* Example: error messages */
.error-message {
  color: #B91C1C; /* darker red for better contrast */
}

/* Example: placeholder */
input::placeholder {
  color: #6B7280; /* target >= 4.5:1 on white */
}{code}
