# MP-315: Category filters do not update the URL (no deep linking)

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Location:* Home page (Dashboard) - category filters  
*Severity:* High  
*Impact:* Users cannot share links to filtered views.

*Description:*
After clicking category filters (e.g., General, Automobile, Retail, Energy, Manufacturing), the URL is not updated. Users cannot:

* Bookmark filtered views
* Share links to specific categories
* Use the browser Back button to return to the previous filter

*Reproduction Steps:*

# Go to the home page (Dashboard)
# Click the “Automobile” category
# Observe the URL remains {{/?mode=default}} (no category parameter)
# Refresh the page — the filter resets

*Evidence:* 

* Screenshot: [dashboard-category-Automobile.png]
* Screenshot: [dashboard-category-Retail.png]

*Recommendation:* 

* Update the URL when the filter changes (e.g., {{/?mode=default&category=automobile}})
* Read URL parameters on page load
* Add support for the History API (pushState)
