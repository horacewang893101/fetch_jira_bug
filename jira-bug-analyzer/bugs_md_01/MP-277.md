**Status:** Backlog

**Assignee:** Unassigned

*Description*
The Key Vault secret {{translator-env}} contains highly sensitive values bundled together, including database credentials and Redis credentials. This monolithic secret increases blast radius and complicates rotation and RBAC.

*Impact*
If {{translator-env}} is compromised, attacker gains full control over DB and Redis, enabling data exfiltration, cache/session hijacking, and persistence.

*Recommended Fix*

* Split into *individual Key Vault secrets* (DB creds, Redis password), rotate all exposed values, enforce Key Vault RBAC per secret, and avoid bundling non-sensitive config with secrets.
* Keep non-sensitive config outside Key Vault.
