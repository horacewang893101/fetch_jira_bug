# MP-324: Login accepts whitespace-only values as valid input

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Location:* Login page / Whole application
*Severity:* Critical  
*Impact:* The user can pass client-side validation with invalid inputs (spaces), which leads to server-side errors and unclear messages.

*Reproduction Steps:*

# Open the login page
# Enter only spaces in the username/login field
# Enter only spaces in the password field
# Submit the form
# Observe that validation treats the values as non-empty

*Evidence:*
Screenshot: [login-empty-inputs.png]
Screenshot: [empty-inputs-validation.png]

*Recommendation:*

* Trim values on the client before validation and submission
* Treat “empty after trim” as missing input (for both login and password)
* Keep validation messages consistent and polite
