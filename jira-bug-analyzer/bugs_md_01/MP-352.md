# MP-352: Incorrect error handling for non-existent agents

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Location:* {{/agent-basic/invalid-uuid-format}}  
*Severity:* High  
*Impact:* A partial page is displayed instead of an error message.

*Description:*
When navigating to an agent page with an invalid UUID, the application attempts to load the page instead of showing an error message. This can lead to an unstable UI state.

*Reproduction Steps:*

# Go to [https://trial.aicentral.top/agent-basic/invalid-uuid-format|https://trial.aicentral.top/agent-basic/invalid-uuid-format]
# The page attempts to load
# There is no clear error message

*Evidence:*

* Screenshot: [error-invalid-agent.png]

*Recommendation:*

* Validate the UUID format before attempting to load
* Show a friendly message like "Agent not found"
* Redirect to the agents list with a notification
