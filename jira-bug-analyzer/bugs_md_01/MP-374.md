# MP-374: Missing container securityContext hardening

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Impacted services:* {{backend}}, {{celery}}, {{frontend}}, {{knowledge}}, {{mcpm}}, {{init-pv-files}}

*Description:*
Containers lack hardening (e.g., {{runAsNonRoot}}, {{readOnlyRootFilesystem}}, {{allowPrivilegeEscalation: false}}, dropping capabilities, runtime-default seccomp).

*Impact:*
Elevated risk of privilege escalation and writable root filesystem abuse if a container is compromised.

*Recommended Fix:*
Add {{securityContext}} per container: enforce non-root execution, disallow privilege escalation, read-only root filesystem, {{seccompProfile: RuntimeDefault}}, and {{capabilities.drop: ["ALL"]}}; reintroduce only minimal required capabilities if truly needed.
