# MP-397: [Recommendation] Knowledge service - Reduce inflated requests

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
Requests: *1000m / 3 GiB*, observed: *~1m / ~235 Mi*.

*Impact:*
Over-reservation reduces scheduling flexibility and increases node pressure/cost.

*Recommended Fix:*

* *CPU request:* 100m
* *Memory request:* 512 Mi–1 GiB
* *CPU limit:* 2000m
* *Memory limit:* 1–2 GiB
* Optional HPA on CPU or app metric.
