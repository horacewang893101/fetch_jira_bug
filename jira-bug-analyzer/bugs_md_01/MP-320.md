# MP-320: Unintuitive navigation in AI Search and AI Reading (Back button behavior)

**Status:** Backlog

**Assignee:** Unassigned

**Description:**
*Location:* Dashboard - AI Search, AI Reading  
*Severity:* High  
*Impact:* Users have difficulty returning to the previous view; browser back navigation can lead to unexpected routing and loss of context.

*Description:*
After opening the AI Search or AI Reading modal, the only obvious way to return to the Dashboard is clicking the X button. Using the browser Back button results in incorrect routing behavior (the user ends up in an unexpected place). This appears to be caused by the {{mode}} URL parameter not being handled correctly in browser history navigation.

*Reproduction Steps:*

# Go to Dashboard
# Click the “AI Search” or “AI Reading” tile
# Observe a modal opens and the URL includes a {{mode}} parameter
# Use the browser Back button
# Observe you do not return to the previous Dashboard state and routing behaves unexpectedly

*Evidence:*

* Screenshot: [ai-search-reading-navigation-issue1.png]
* Screenshot: [ai-search-reading-navigation-issue2.png]
* Screenshot: [ai-search-reading-navigation-issue3.png]

*Recommendation:*

* Correctly handle {{popstate}} for the {{mode}} parameter
* When closing via Back, restore the previous URL state (remove {{mode}}) and update UI state accordingly
* Consider using {{history.replaceState}} instead of {{history.pushState}} for modal open/close to avoid polluting history
* Alternatively, avoid modifying the URL for AI Search/AI Reading modals
